# 10 写作流程与质检清单

## 全流程概览

```
Step 0  一句话验证
  ↓
Step 1  选题（3 方案 → 用户选择）
  ↓
Step 2  章节大纲（8-15 章 + 情绪过山车）
  ↓
Step 3  逐章写作
  ↓
Step 4  合并 + 字数核验
  ↓
Step 5  人味打磨
  ↓
Step 6  梗概撰写
  ↓
Step 7  终检 + 投稿/发布
```

---

## Step 0：一句话验证

动笔前，故事概念必须同时满足：

| 条件 | 检验 |
|------|------|
| 有悬念 | 听完想知道后面 |
| 有反常 | 违反日常认知 |
| 有画面 | 脑中能成像 |

**三个缺一 → 不写。**

---

## Step 1：选题（输出 3 方案）

每个方案包含：

```markdown
- 标题（盐选公式）
- 300-500 字梗概
- 主角人设（职业、缺陷、说话风格示例）
- 核心反转（一句话）
- 免费部分结尾断点（具体场景）
- 与同类差异化点
```

**用户确认后才进入 Step 2。**

---

## Step 2：章节大纲

### 输出格式

```markdown
第 X 章：[章节标题]（目标字数：XXXX）
- 功能：钩子/铺垫/升级/反转/爽点/收束
- 内容摘要：2-3 句话
- 结尾钩子：本章最后留什么悬念
- 关键场景/对话：必须包含的场景
```

### 硬性标注

- [ ] 付费墙位置（第 3-4 章）
- [ ] A 级反转位置（60-75%）
- [ ] 编辑审核线（前 4000 字）
- [ ] 各章目标字数（总和 ≥ 13000）

---

## Step 3：逐章写作

### 写作规则

1. 按章节顺序，每章独立完整
2. 每章写完统计字数，**未达标必须扩充**
3. 叙述者语气全文一致
4. 每章结尾有钩子（最后一章除外）

### 字数不够时的扩充方法

✅ 增加对话  
✅ 加入 1-2 个感官细节  
✅ 补充简短内心活动  
✅ 增加与主线相关的小场景  

❌ 灌水重复描写  
❌ 无意义景物铺垫  

### 前 4000 字（编辑生死线）额外要求

- 第一句有异常感
- 禁止景物开头
- 前 500 字：角色 + 问题 + 为什么特别
- 前 2000 字：让读者关心主角
- 3000-4000 字：不可能不往下看的悬念

### 付费部分（第 4 章起）额外要求

- 每 800 字一个情绪高潮
- 禁止继续铺垫
- A 级反转在 60-75%

---

## Step 4：合并与字数核验

```markdown
| 章节 | 目标字数 | 实际字数 | 达标？ |
|------|----------|----------|--------|
| 1 | 1500 | | |
| ... | ... | | |
| **合计** | ≥13000 | | |
```

- 任何章节低于目标 80% → 回炉
- 全文必须 ≥ 12000 字

---

## Step 5：人味打磨

| 检查项 | 标准 |
|--------|------|
| 句式 | 连续 5 句节奏相似 → 打散 |
| 对话 | 遮住名字能分辨角色 |
| 闲笔 | 全文 ≥ 5 处生活化闲笔 |
| 情绪 | 有起伏，非全程同一强度 |
| 开头 | 重读前 4000 字 —— 编辑会继续吗？ |
| AI 味 | 禁用句式清零 |
| 标题 | 符合盐选公式 |
| 梗概 | 300-500 字已准备 |

### 扩充技巧

- 动词替换：「推开」→「踹开」
- 加入气味、触感
- 个性化吐槽
- 故意制造段落长短不均

---

## Step 6：梗概撰写

```markdown
## 投稿梗概（300-500 字）

**类型**：
**核心设定**：
**主角**：
**主线**：
**核心反转**：（可剧透）
**结局**：
```

要求：简洁专业，编辑 10 秒判断商业价值。

---

## Step 7：终检清单

### A. 结构终检

```
□ 一句话说清主线
□ 8-15 章，每章 800-2500 字
□ 全文 ≥ 12000 字
□ 前 4000 字 ≥ 3 钩子
□ 付费墙第 3-4 章
□ A 级反转 60-75%
□ 每 600-800 字有转折
□ 每章结尾有钩子
```

### B. 文风终检

```
□ 第一人称
□ 段落 1-3 句
□ 独句段 ≥ 30%
□ 对话 40-50%
□ 无 4 句以上段落
□ 禁用句式清零
□ 朗读顺口
```

### C. 情绪终检

```
□ 惊→疑→憋屈→爽→余韵
□ 主角有能动性
□ 憋屈后强反弹
□ 结尾不升华
```

### D. 逻辑终检

```
□ 时间线自洽
□ 人物一致
□ 无「为什么不直接」漏洞
□ ≥ 2 次「差点暴露」
□ 盟友有代价/犹豫
```

### E. 合规终检

```
□ 无题材红线
□ 有差异化
□ 非连续同模式第 3 篇
□ 梗概已备
```

## 智能体工作模式建议

### 模式 1：全流程创作

用户给模糊想法 → 3 方案 → 选方案 → 大纲 → 逐章 → 终检

### 模式 2：润色魔改

用户提供干巴段落 → 按 05 文风规范「场景再造」

### 模式 3：拆文学习

用户提供爆款 → 按 09 模板拆文 → 提取模板 → 生成原创方案

### 模式 4：卡点设计

用户提供前 3 章 → 设计 5 种付费墙截断方案


## 质量门禁（不可跳过）

| 门禁 | 条件 | 不通过则 |
|------|------|----------|
| G1 | 一句话验证 | 回到 Step 0 |
| G2 | 用户确认方案 | 不可写大纲 |
| G3 | 大纲标注付费墙/A级反转 | 不可开写 |
| G4 | 每章字数达标 | 不可合并 |
| G5 | 终检清单全过 | 不可输出终稿 |
