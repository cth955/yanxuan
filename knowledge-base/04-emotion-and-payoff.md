# 04 情绪曲线与爽点设计

## 核心原则

> 能卖钱的故事不是「写」出来的，是「设计」出来的。

大纲 = **情绪过山车设计图**，不是「第一段写什么第二段写什么」的流水账。

---

## 情绪曲线：爆款模式

### 标准曲线

```
惊 → 疑 → 怒/憋屈 → 爽/反转 → 余韵
```

### 反模式（跑不动推荐）

```
疑 → 揭秘 → 感动 → 泪
```
纯催泪在盐选推荐流里**几乎跑不动**——没有悬念钩子，算法不推。

---

## 情绪落差：弹簧法则

> 压得越狠，弹得越高。

| 阶段 | 读者情绪 | 写作动作 |
|------|----------|----------|
| **憋屈期** | 愤怒、心梗、替主角不值 | 主角被踩到泥里，被所有人看不起 |
| **反弹期** | 核弹级爽感 | 十倍百倍还回去 |

### 新手致命错误

❌ 不敢虐主 —— 主角一直顺风顺水  
❌ 只有爽没有憋屈 —— 爽感贬值  
❌ 憋屈太久不反弹 —— 读者弃文

---

## 六大爽点类型（按有效性排序）

| 排名 | 类型 | 机制 | 示例 |
|------|------|------|------|
| 1 | **复仇打脸** | 恶人遭报应 | 手撕绿茶、公开处刑 |
| 2 | **认知碾压** | 你不懂的我一眼看穿 | 扮猪吃虎、身份曝光 |
| 3 | **资源降维** | 背景/财富/人脉压制 | 穷小子其实是继承人 |
| 4 | **情感救赎** | 被辜负后的被珍惜 | 追妻火葬场（男难受） |
| 5 | **秘密揭露** | 真相大白 | 层层剥洋葱 |
| 6 | **暧昧拉扯** | 欲拒还迎 | 误会/吃醋/意外接触 |

### 爽点组合链

```
被打脸 → 暗中布局 → 反向碾压 → 对方求饶
```

建议组合使用，形成**情绪链**而非单点爆发。

---

## 三大底层引擎

### 1. 信息差

> 玩弄信息差 = 玩弄读者的心。

| 类型 | 说明 |
|------|------|
| 读者 > 主角 | 读者猜到了，主角还不知道 → 读者觉得自己聪明 |
| 作者 > 读者 | 推翻读者猜测 → 「卧槽还能这样」 |
| A > B | 角色间信息不对等 → 戏剧张力 |

**悬疑/职场/婚姻**类必用。

### 2. 人设钩子

> 完美的人没人爱看。

| 手法 | 示例 |
|------|------|
| 反差萌 | 女杀手喜欢养多肉 |
| 缺陷 | 表面软实际狠 |
| 矛盾特质 | 霸道总裁怕老婆 |
| 能动性 | 主角「做」了什么，而非只「发现」 |

**主角必须有能动性**：
- ❌ 只发现真相 → 太被动
- ✅ 发现 + 布局 + 反击 → 有行动力

### 3. 目标 + 阻碍

每 2 章左右一个循环：

```
主角目标 → 意外阻碍 → 新策略 → 小胜利/小挫败
```

## 爽点节奏表（实操工具）

### 制作方法

| 字数段 | 爽点类型 | 强度(1-5) | 读者情绪 |
|--------|----------|-----------|----------|
| 0-800 | 钩子/冲突 | 4 | 惊 |
| 800-1600 | 憋屈加深 | 3 | 怒 |
| 1600-2400 | 小反击 | 4 | 爽 |
| 2400-3200 | 假胜利 | 3 | 松→紧 |
| 3200-4000 | **付费墙悬念** | 5 | 抓心 |
| ... | ... | ... | ... |

### 理想分布

- **波峰波谷交替** —— 避免连续高潮导致麻木
- **第 1/3 处**安排小高潮预热 —— 前半程不能太平
- 每 700 字至少一个**记忆点**

---

## 情绪升温点

单位字数内建议埋设 **3 个情感升温点**（甜/虐/燃/怕均可）。

### 发糖 vs 捅刀

| 时机 | 作用 |
|------|------|
| 发糖 | 让读者觉得甜，后面捅刀更疼 |
| 捅刀 | 制造紧张，后面反弹更爽 |
| 绝路 | 读者觉得「完了死定了」 |
| 反击 | 前面憋屈的十倍偿还 |

---

## 人物设计：好人有暗面，坏人有底色

| 角色 | 盐选偏好 |
|------|----------|
| 主角 | 有缺陷、有秘密、有反差 |
| 反派 | 不要纯恶 —— 最好有悲情底色或合理动机 |
| 配角 | 有独特说话 DNA，不能全员工具人 |

**立人设的细节法**：
- ❌ 「他很穷」
- ✅ 泡面吃完了，汤舍不得倒，打个鸡蛋又是一顿

---

## 结尾情绪

| 要求 | 说明 |
|------|------|
| 有释放 | 憋屈得到回应 |
| 干脆 | 故事讲完就停 |
| 不升华 | 禁止说教、人生感悟 |
| 可留余韵 | 最大秘密可在松弛尾声中泄露 |

---

## 智能体情绪设计自检

```
□ 大纲是否是「情绪过山车」而非「事件流水账」？
□ 是否有足够的憋屈期（至少 2 处）？
□ 反弹是否强过憋屈（强度匹配）？
□ 主角是否有主动行动（非纯被动发现）？
□ 每 600-800 字是否有一个转折/爽点？
□ 是否有 1 个 A 级 + 2 个 B/C 级反转？
□ 结尾是否干脆、不升华？
```
